# Deployment & local development

Instructions for running Utasync on your machine and deploying the static build. For a product overview and feature list, see the [README](../README.md).

## Prerequisites

| Requirement | Notes |
|---|---|
| **Node.js 18+** | Required for Vite 8 and the build toolchain. |
| **npm** | Comes with Node; other package managers work if you prefer. |
| **Modern browser** | Chrome, Edge, or Safari recommended. WebGPU and sufficient RAM improve auto-align performance. |
| **Disk space** | ~300 MB+ for cached AI models (Whisper, embeddings) after first use; additional space for uploaded audio in the browser. |

Utasync is a client-only PWA. There is no backend server to run.

## Installation

```bash
git clone https://github.com/Ninjaruss/utasync.git
cd utasync
npm install
```

### Japanese tokenizer dictionary

Japanese tokenization uses [kuromoji](https://github.com/takuseno/kuromoji). Copy its dictionary into `public/dict` so the dev server and production build can serve it:

```bash
cp -r node_modules/kuromoji/dict public/dict
```

Re-run this after `npm install` if you remove `public/dict`.

### Optional: vocal separation model

On capable devices (WebGPU + 6 GB+ RAM), auto-align can run vocal separation before transcription. Place the ONNX model at:

```
public/models/demucs-v1.onnx
```

If the file is missing, alignment still works on the **lite** path (transcription only). The app detects device tier automatically.

### AI models (automatic download)

Whisper (`Xenova/whisper-small`) and embedding models download from Hugging Face the first time auto-align or word alignment runs. They are cached in the browser (Cache Storage, ~30-day retention). A network connection is required for the initial download only.

## Environment variables

Utasync does not use `.env` files for runtime configuration. The app reads `import.meta.env.DEV` for development-only UI.

For production licensing, replace the placeholder LemonSqueezy public key in `src/payment/license.ts` with your store's ES256 public key before shipping.

## Run locally

```bash
npm run dev
```

Open the URL Vite prints (typically `http://localhost:5173`). The dev server includes middleware that serves kuromoji `.dat.gz` dictionary files as raw bytes (required for correct decompression in the browser).

## Build

```bash
npm run build
```

Output goes to `dist/`. Preview the production build locally:

```bash
npm run preview
```

## Deploy

Utasync builds to static files suitable for any static host (Vercel, Netlify, Cloudflare Pages, S3 + CDN, etc.).

1. Run `npm run build`.
2. Upload the contents of `dist/` to your host.
3. Ensure **`/dict/*.gz`** is served without automatic gzip re-encoding. Some static hosts treat `.gz` files as pre-compressed and set `Content-Encoding: gzip`, which breaks kuromoji loading. Serve them as `application/octet-stream` with no content encoding, or configure your host to disable double compression for `/dict/`.
4. Configure SPA fallback so unknown routes serve `index.html` (standard for Vite SPAs).
5. HTTPS is required for PWA install, OPFS, and service worker registration.

The PWA manifest and Workbox service worker are generated by `vite-plugin-pwa`. ONNX models under `/models/` use cache-first runtime caching when present.

### Production checklist

- [ ] `public/dict` populated (kuromoji)
- [ ] `public/models/demucs-v1.onnx` present if you want vocal separation on full-tier devices
- [ ] LemonSqueezy public key updated in `src/payment/license.ts`
- [ ] HTTPS enabled
- [ ] Dictionary gzip serving verified on your host

## Lint and test

```bash
npm run lint
npx vitest run
```

Vitest is configured in `vite.config.ts` with a jsdom environment. Run a single file with `npx vitest run tests/path/to/file.test.ts`.

## Browser support

| Feature | Requirement |
|---|---|
| Core player & YouTube mode | Modern evergreen browser |
| Local audio + OPFS | Chromium-based or Safari 17+ |
| Auto-align (Whisper) | WebGPU or WASM fallback; 4 GB+ RAM recommended |
| Vocal separation | WebGPU + 6 GB+ RAM; demucs model file |
| PWA install | HTTPS + service worker support |

## Troubleshooting

**Japanese readings or tokenization fail** — confirm `public/dict` exists and `/dict/` URLs return raw gzip bytes (see deploy notes above).

**Auto-align stuck on "Loading model"** — check network access for the first Hugging Face download; clear site data and retry if cache is corrupted.

**No vocal separation step** — device may be on the lite or manual tier; add the demucs ONNX file and use a WebGPU-capable browser with enough memory.

**Storage warnings** — audio and models live in the browser; use Settings to review usage and remove songs if quota is low.
